## Project Overview
Goals and objectives
- Build a production-ready modern frontend for a comprehensive business website demonstrating:
  - Responsive layout across breakpoints
  - Accessible UI components (WCAG AA)
  - Clean component-based architecture (reusable components)
  - Client-side routing, state management for interactive screens
  - Build and deployment configuration for fast static hosting
- Audience: instructors and reviewers assessing modern frontend best practices.

Success criteria
- App builds without errors.
- Core pages (Home, About, Services, Contact, Project showcase) render correctly on desktop and mobile.
- Component tests and basic end-to-end validation present.

## Setup (step-by-step)
Prerequisites
- Node.js 18+ (or LTS), npm or yarn
- Git

Local installation
1. Clone repository:
   - git clone https://github.com/bharatdasari0528/Comprehensive-Business-Website.git
2. Change into the folder:
   - cd "Master Modern Frontend Development"
3. Install dependencies:
   - npm install
   - or yarn

Development
- Start dev server:
  - npm run dev
  - or yarn dev
- Open http://localhost:3000 (or port printed in console)

Build for production
- npm run build
- npm run start (or serve the `dist` / `build` folder via static host)

Notes
- If the project uses a specific framework (React / Next.js / Vite / Vue), replace scripts above with framework-specific commands. See package.json for exact scripts.

## Technical details
Architecture & patterns
- Framework / Tooling: State which you used (React/Vue/Svelte/Next/Vite). Example decisions:
  - Component-first design with atomic components
  - CSS strategy: CSS Modules / Tailwind / Styled Components
  - State management: local component state + context / small global store (Zustand / Redux) if needed
  - Routing: client-side router (React Router / Vue Router / Next.js file-based)
- Data flow
  - Top-down props for static sections
  - Context or central store for app-level state (auth, cart, theme)
- Algorithms & data structures
  - No heavy algorithms required — where used:
    - Debounced search input (debounce using timer)
    - Client-side filtering and sorting for project list (Array.filter, Array.sort)
    - Lazy-loading images (IntersectionObserver)
  - Use arrays and objects to represent lists and records (projects, services)
- Performance
  - Code-splitting and lazy-loading routes
  - Optimize images (use srcset / webp)
  - Use cache headers in deployment

Security & accessibility
- Escape and sanitize dynamic content.
- All interactive elements keyboard accessible, semantic HTML, aria labels for complex widgets.

## Testing
Types of tests and examples
- Unit tests
  - Frameworks: Jest + Testing Library (React Testing Library), or Vitest
  - Example test case (pseudocode):
    - Render Header component -> expect logo alt text present -> expect navigation links visible
- Integration
  - Simple test: render Contact form -> fill inputs -> submit -> assert success message
- Manual validation checklist
  - Navigation works on keyboard (Tab focus)
  - Forms validate and show errors
  - Layout matches screenshots across breakpoints (mobile, tablet, desktop)
  - Lighthouse audit: aim for >90 performance/accessibility/SEO

Sample unit test (pseudocode)
```js
test('renders hero title and CTA', () => {
  render(<Hero title="Welcome" />);
  expect(screen.getByRole('heading', { name: /welcome/i })).toBeInTheDocument();
  expect(screen.getByRole('link', { name: /get started/i })).toHaveAttribute('href', '/contact');
});
```

## Component architecture
Component hierarchy and data flow (example)
- App
  - Layout
    - Header
      - Logo
      - Nav
    - MainRouter / Pages
      - HomePage
        - Hero
        - ServicesGrid
          - ServiceCard (reusable)
        - ProjectsPreview
          - ProjectCard
      - AboutPage
      - ContactPage
        - ContactForm
    - Footer

Mermaid-like diagram (ASCII, paste into mermaid-compatible viewers if needed)
```text
App
└─ Layout
   ├─ Header
   │  ├─ Logo
   │  └─ Nav
   ├─ MainRouter
   │  ├─ HomePage
   │  │  ├─ Hero
```

Data flow
- Top-level App provides theme and user context
- Page components fetch or receive data and pass to child components via props
- User interactions bubble up via callback props or context dispatch
